# Terms and Conditions Template

## For Epic Portal Registration

Epic requires a **secure HTTPS URL** where users can review Terms and Conditions before authorizing access to their health data.

## Requirements

✅ **Must be HTTPS** (secure connection)  
✅ **Must be publicly accessible** (no login required)  
✅ **Should be mobile-friendly** (responsive design)  
✅ **Should be in plain language** (patient-friendly)  
✅ **Should cover data access and privacy**

## Quick Setup Options

### Option 1: GitHub Pages (Free, Easy)
1. Create a repository on GitHub
2. Add an `index.html` file with your Terms
3. Enable GitHub Pages in repository settings
4. Your URL will be: `https://yourusername.github.io/your-repo-name/terms`

### Option 2: Your Own Domain
1. Host a simple HTML page at: `https://yourdomain.com/terms`
2. Ensure SSL certificate is valid (HTTPS required)
3. Make it publicly accessible

### Option 3: Static Site Hosting
- **Netlify**: Free tier, easy deployment
- **Vercel**: Free tier, simple setup
- **Firebase Hosting**: Free tier available

## Example URL Format

```
https://yourdomain.com/terms
https://mychartfhir.app/terms-and-conditions
https://yourusername.github.io/mychart-terms/terms
```

## What Epic Will Check

Epic's review will verify:
- ✅ URL is accessible (no 404 errors)
- ✅ Uses HTTPS (secure connection)
- ✅ Content is appropriate for patient-facing app
- ✅ Clearly explains data access and use
- ✅ Mentions privacy and security measures
